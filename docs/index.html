Why this repository exists

Generic frontend agents usually fail at complex website tasks. They modify code before inspecting, treat mobile as a cropped desktop, break ScrollTriggers, ignore accessibility, and claim completion without production verification. This repository solves these failures by dividing website engineering into focused, specialist skills.

Common Agent Failures

Generic agents skip inspection

They rush to patch style files without understanding CSS token hierarchies, breaking global design architectures.

Mobile becomes cropped desktop

They shrink container dimensions or hide desktop animations, resulting in long vertical lists and unreadable layouts.

Animation ships without verification

They deploy timelines that trigger layout shifts or run slow scroll calculations, causing runtime stutter.

Local and production drift apart

They assume local build compilation means the live server state behaves identically, ignoring caching or env config errors.

The Repository Solution

Specialized Roles

Focused skills for layout reconstruction, mobile adaptation, scroll-animations, SEO, accessibility, and deployments.

Explicit Workflows

Structured sequences that route assignments systematically from mapping to coding to QA to delivery gates.

Measurable Acceptance Criteria

Strict requirements specifying minimum touch targets, no scroll overflow, and exact viewport screenshot matrices.

Evidence-Based Completion

Completion reports require files changed, live URLs, and screenshots of start, middle, and final scroll states.

How to use a skill

Copy the prompt template below when assigning website tasks to your AI coding agents. Replace the template markers with your project specifics.

Use the <skill-name> skill. PROJECT <project name> TARGET <route, component, file, or URL> CURRENT PROBLEM <exact failure> DESIRED RESULT <observable behavior> CONSTRAINTS - Do not modify: - Preserve: - Existing libraries: - Required breakpoints: REQUIRED TESTS - Breakpoints: - Input methods: - Routes: - Reduced motion: REQUIRED REPORT - Root cause - Files changed - Screenshots - Remaining limitations

How to create a new skill

Expand the repository capabilities by adding custom specialist skills following this workflow:

1

Copy template

Copy the SKILL_TEMPLATE.md from the templates folder.

2

Create folder

Create `agents/<new-skill-name>/` utilizing lowercase kebab-case naming.

3

Add files

Create `SKILL.md` (mission, guardrails, criteria), `metadata.json` (categories, use-cases), and `examples.md` files.

4

Validate and build

Update the manifest, run `npm run validate` to test schemas, and `npm run build:docs` to update this website.

Repository structure

Click on any folder in the tree below to expand details about its contents and architectural purpose in the repository.

  • Contains the 24 specialized agent skills. Each subdirectory contains a `SKILL.md` file (defining trigger conditions, workflow, deliverables, and handoff recommendations), a `metadata.json` file (specifying when/avoid cases, inputs, and companion skills), and an `examples.md` file (with prompts).
  • Holds multi-agent workflows including Full Website Audit, Mobile Reconstruction, and industry verticals. Includes the `workflow-index.json` which maps all steps and sequences.
  • Houses specific checklists for accessibility, animation, performance, SEO, mobile adaptation, and release gating criteria to guarantee visual and functional quality.
  • Reusable templates for agents, including new skill formats, request templates, website audits, and completion report templates.
  • Machine-readable JSON schemas ensuring that skills, workflows, audits, and completion reports conform to structured formats.
  • Case study directories containing reference plans, primary risks, and prompts for real websites.
  • The codebase for this interactive documentation portal, serving structured HTML, CSS, JavaScript, and generated JSON files offline and on GitHub Pages.
  • Node.js scripts used to validate skill definitions against schemas, verify internal links, build documentation JSON data, and regenerate indices.
  • GitHub actions workflow definitions for running continuous integration validations and deploying static files to GitHub Pages, plus issue and pull request templates.
